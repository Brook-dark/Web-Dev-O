Specificity and cascade

Specificity:

Specificity determines which one of the css property will be applied to an element when there are multiple properties applied.Specificity is determined by cascade algorithm.

Cascade Algorithm:

CSS stands for cascading style sheets. The cascade is the algorithm which is used to solve conflicts in which multiple css rules are applied. It helps us to determine which one of the rule will take effect despite other rules.

Algorithm :

  • Position and order of appearance: the order in which your CSS rules appear
  • Specificity: an algorithm that determines which CSS selector has the strongest match
  • Origin: the order in which CSS appears and where it comes from, whether that is a browser style, CSS from a browser extension, or your authored CSS
  • Importance:some CSS rules are weighted more heavily than others, especially with the !important rule type.

Lets look into all these one by one :

Position and order of appearance:

When two rules have the same specificity, the last one defined will take precedence. This is why it is important to define your CSS rules in the correct order.

Specificity

Each css rule has specific specificity.Simply said, each css rule has a specific number which makes it more specific than others and any style assigned to that property will override other elements with low specificity.Here is the list by rank:

  1. Inline styles
  2. ID
  3. Class, pseudo-class, attribute
  4. Element, pseudo-element
  5. Universal selector

Inline Style > ID Selector > Class or Attribute Selector > Element Selector > Universal Selector.

Calculation of specificty :

To calculate specificity, following are the values assigned to each part of the selector.

  • Universal Selector: 0
  • Element selectors and pseudo-elements: 1
  • Class selectors, attribute selectors, and pseudo-classes: 10
  • ID selectors: 100
  • Inline styles: 1000

Quiz :

What will be the specificity value of the following selector:

a.superclass.one-class[href]:hover { color: red; }

Answer: 41

Solution:

  • Element selector: 1
  • Class selector: 10
  • Class selector: 10
  • Attribute selector: 10
  • Pseudo-class: 10

Add up all these values: 1 (element) + 20 (classes) + 10 (attribute) + 10 (pseudo-class) = 41.

Importance

Some CSS rules are weighted more heavily than others, especially with the !important rule type. The !important rule will override all other styles, regardless of specificity. It is important to use !important sparingly, as it can make your CSS difficult to manage and maintain.

It has an specificity of 10000

Origin

Origin is the order in which CSS appears and where it comes from, whether that is a browser style, CSS from a browser extension, or your authored CSS. The order of origin is as follows:

  • Browser styles
  • Browser extensions
  • Author styles
  • !important

Author styles will override browser styles, and !important will override all other styles.
