Position Property

Explanation:

The position property specifies the type of positioning method used for an element.

There are five different position values:

  • static
  • relative
  • fixed
  • absolute
  • sticky

Elements are then positioned using the top, bottom, left, and right properties. However, these properties will not work unless the position property is set first. They also work differently depending on the position value.

Static(Default)

Explanation:

It is the default value for position.Elements with this property will show as they are placed like if a element occurs first it will occur first and if one occurs afterwards it will occur afterwards.Think of it like the normal way things are placed in a website unless you change it with other position properties.Its syntax is like position:static; but as it is the default value you dont need to set this value unless you want to remove certain position effects.This property has some limitations.Which are: You cant use any property related to position property if the value of position property is set to 0.These properties can be top,bottom,left,right,z-index.These properties will only work if the position property's value is set to something aside from static.

Relative

Explanation:

With relative property you can change the position of a container or element using offset properties{top,bottom,left,right} and also use z-index, which i will explain below.Important part is that it will reserve the space for element meaning no other element will over take the place the place of the container or element displaced.This property will also be useful when we will use absolute property but we will discuss that later.

Z-index:

Z-index decides which element will stack over the other when any type of positioning is used other than static positioning or default one with offset properties{top,bottom,left,right}.The element or container with higher z-index value will stack over the one with lower z-index value.Some important things to note about z- index:

  • Z-index will only work if position property is set to any other value aside from static value or simply said you will have to assign a position property with value other than static value for z-index to work.
  • Z-index value should be greater than 0 as its the default value and has no effect but also dont use very large values for stacking elements or containers over other as it may lead to z-index not working.
  • When elements have the same stacking context and no explicit z-index values, the browser stacks them in the order they appear in the HTML source code.
  • Elements that appear later in the HTML are stacked on top of elements that appear earlier.

For more information on z-index visit: z-index .Experiment with values on the website to grasp the concept.

Offset properties:

These are properties which are used to modify position of a element or container.It has 4 values described below but at one time you can use only 1 of the two pairs like you can use one of top or bottom at once.Similarly you can use one of left or right at once.These values are:

  • top:Moves element down.
  • bottom:Moves element up.
  • left:Moves element left.
  • right:Moves element right.

Absolute

Explanation:

Absolute positioning is like relative positioning but with slight differences.These differences are explaned below:

  • Purpose: It removes an element from the normal document flow means if we use a offset property then when it gets displaced the elements below it will take its space and will stack above them(unless you have used a z-index value on other element).Aside from this it positions an element relative to its nearest positioned element meaning if we used a offset property then it will take effect according to the nearest element with any position value aside from static.If the nearest element doesnt have a assigned position or its position value is static then it will position itself according to the html or root tags.I will a example demonstrating it.
  • Offset properties: In absolute positioning offset properites work in quite opposite way like if you assign 20px bottom then instead of moving up the element will move downwards.Its because of the reference points as in absolute positioning the reference is according to its positioned ancestor so in absolute positioning the element will move up from the bottom of the element.So its effect may seem opposite.
  • Margin collapse Margins dont collapse in absolute positioning.This is one the main differences between relative and absolute positioning.Margin collapse only work when elements are in the normal document flow meaning they are either in static positioning or relative positioning so their margins will collapse or the margin whose value is greater will take effect but its not the case in absolute positioning as the absolutely positioned element is taken out of the normal document flow so its margins will not collapse with other elements margins instead it will stay separate.

Use-cases:

Absolute positioning is used for various purposes.Some of them are:

  • Overlays, dropdown menus, or tooltips that need precise placement.
  • Elements that should ignore document flow (e.g., a fixed sidebar).

Differences between relative and absolute positioning:

Relative VS Absolute
Aspect Relative Positioning Absolute Positioning
Document Flow Stays in flow (original space reserved) Removed from flow (no space reserved)
Positioning Context Relative to its normal position Relative to nearest positioned ancestor
Margin Collapsing Margins collapse with adjacent elements Margins don't collapse with flow elements
Use Cases Minor adjustments (e.g., nudging elements) Precise placement (e.g., modals, tooltips)
Ancestor Dependency No dependency Requires positioned ancestor (e.g., relative, fixed)

Fixed

Explanation:

Fixed positioning is another positioning value that is used to make a element or container become attached to your screen its like someone has nailed it there, just like absolute positioning the fixed element is also taken out of the normal document flow. Some key behaviours of fixed positioning are:

  • Document flow: The element is completely removed from the document flow meaning its space is not reserved in the document.
  • Position: Its position relative to the viewport meaning that its not positioned relative to any ancestor positioned element or parent element like absolute posiioning instead its positioned relative to the viewport size of the web window and offset properties take effect on it according to it.
  • Scroll: It remains in the exact same position even when the page is scrolled meaning scrolling past it wont hide it(you'll see it in the example).
  • Boundries: It ignores the parent containers boundries meaning if you scroll past a fixed positioned element it will ignore the boundries of its parent and push past it.
  • Other elements: Other elements will flow as if there wasnt any fixed element because it is taken out of the normal document flow.
  • Offset properties: You position it using offset properties.

Usecases:

Some common purposes for which fixed positioning is used are:

  • Navigation bars that stay on top of the screen as you scroll.
  • Cookie activition reminders that will stay there until you accept or decline.
  • Chat widgets that will stay in the screen for ease of use.
  • Modal overlays that stay centered.
  • Back to top buttons that are always accessible.
